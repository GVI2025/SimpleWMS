# Groupe X – Tests de charge HTTP automatisés

## Objectif
Concevoir un test de charge HTTP simple sur l’API, le lancer localement, et si possible l’intégrer à une GitHub Action.

## Travail attendu

### 1. Mise en place locale
- Choisir un outil : `k6`, `locust`, `wrk`, `autocannon`...
- Simuler un scénario simple (ex : envoi répété de requêtes POST ou GET)
- Mesurer : temps de réponse, débit, erreurs

### 2. Intégration CI (optionnel)
- Ajouter un test rapide dans une GitHub Action

### 3. Présentation attendue
- Lancer une démo locale du test
- Montrer les résultats et les interpréter

**Livrables :**
- Script de test de charge
- Démo ou capture des résultats
